1. Executive summary

What this is

macrogen-es.com is the public-facing website for Macrogen Ibero-America — the joint operation of two official Macrogen Inc. branches:

  • Macrogen Spain (Madrid, established 2016) — serves Spain and Portugal
  • Macrogen Chile (Santiago, newest branch) — serves Chile and Peru

The site presents the regional unit as one brand under the umbrella of Macrogen Inc. (Seoul HQ) while preserving the local identity of both branches.

Why we built it the way we did

The architectural goal was maximum simplicity, maximum control, near-zero recurring cost:

  • No CMS, no database, no backend → no security patches, no server bills, no downtime risk
  • Static HTML, CSS, vanilla JS → editable by anyone with a text editor, deployable anywhere, indestructible
  • Git-based content workflow → every change is versioned and reviewable; rollback is instant
  • Free-tier hosting on Vercel → global CDN, automatic HTTPS, ~$0/month forever for our traffic level

4. Technology stack & rationale

Why static HTML + CSS + vanilla JS

DecisionRationale
No framework (React, Vue, Svelte)Site is content-driven, not interactive app. Frameworks add complexity, build steps, and dependencies for zero functional benefit.
No CMS (WordPress, Contentful)No database = no security patches, no $30/month hosting, no plugin breakage.
No build tools (Webpack, Vite)Source = production. What you write is what ships.
No CSS framework (Tailwind, Bootstrap)Custom CSS is ~25 KB and gives full design control.
Vanilla JS onlyTotal JS is <100 lines. No need for jQuery or libraries.

Color rule: green vs pink CTAs. We deliberately split CTAs into two color families:
  • Green = "Get a quote / talk to us"
  • Pink = "Order directly (skip the quote)"
Apply consistently — never use green for "Order Now" or pink for "Get quote".

Typography

  • Font family: Inter (Google Fonts) — modern, scientific, excellent at small sizes
  • Weights: 300, 400, 500, 600, 700, 800, 900
  • Type scale: H1 auto-scales by viewport, body 16px line-height 1.6

Responsive breakpoints

380-600px Phone (hamburger nav, single column)
<380px    Tiny phone (max compression)

7. Form logic & smart country routing

The flow

  1. Visitor lands on contacto.html
  2. First form field is a required country dropdown
  3. They select their country (Spain / Portugal / Chile / Peru / Otro)
  4. JavaScript updates 3 things in real time:
    1. Email field placeholder → country-specific TLD (e.g., tu@laboratory.cl)
    2. Phone field placeholder → country-specific code (e.g., +56 9 ...)
    3. Visible info box → shows which inbox will receive the request
  5. Visitor fills the form, hits "Enviar Solicitud"
  6. (Currently demo) Friendly thank-you message appears
    (Future, with Formspree) Email is sent to info-spain@ or info-chile@

Routing matrix

SpainMadridinfo-spain@macrogen.com
PortugalMadridinfo-spain@macrogen.com
ChileSantiagoinfo-chile@macrogen.com
PeruSantiagoinfo-chile@macrogen.com
Other countryMadrid (default)info-spain@macrogen.com
Current state (demo): Form has data-demo attribute. On submit, JavaScript shows "thanks" message but doesn't send anywhere. To activate real lead capture: sign up at formspree.io, get unique form endpoint URL, replace form's action attribute, remove data-demo.

8. Deployment pipeline

The full chain (Edit → Live, ~1 minute)

Step 1: Edit on your computer
        Open any .html or .css file in a text editor
        Make changes, save the file
        ↓
Step 2: Commit & push via GitHub Desktop
        Open GitHub Desktop app
        Type a commit message
        Click "Commit to main" → "Push origin"
        (~5 seconds)
        ↓
Step 3: Vercel auto-deploys
        Vercel sees the GitHub push within 5 seconds
        Builds (no build needed for static HTML — instant)
        Deploys to all CDN edge locations globally
        (~30 seconds total)
        ↓
Step 4: Live for visitors
        macrogen-es.com serves the new version
        Old version preserved as instant rollback

Rollback procedure

  1. Go to vercel.com/dashboard → click macrogen-es project
  2. Click "Deployments" tab
  3. Find the last working deployment (status: "Ready")
  4. Click the "⋯" menu → "Promote to Production"
  5. Live within seconds

9. Day-to-day update workflow

Update existing copy (text changes)

  1. Open File Explorer → Documents/Claude/Projects/Macrogen-es/
  2. Open the relevant .html file in a text editor
  3. Use Find (Ctrl+F) to locate the text you want to change
  4. Edit the text — keep HTML tags intact, only change text between them
  5. Save (Ctrl+S)
  6. Open GitHub Desktop → commit + push

Add a new blog post

  1. Copy an existing blog file (e.g., blog/que-es-ngs.html) and rename
  2. Update the title, meta description, header content
  3. Replace the article body content
  4. Open blog.html and add a new card pointing to your new post
  5. Commit + push

Update brand colors site-wide

  1. Open assets/css/main.css
  2. Find the :root block at the top
  3. Change the hex values
  4. Save → commit + push
  5. Every page updates automatically

Replace a logo

  1. Save the new logo file as logo.png in assets/img/ (overwriting)
  2. Commit + push
What NEVER to touch: the .git/ folder (hidden, GitHub Desktop manages it), the <head> section of HTML files unless you understand <meta>, and the <script> / <style> tags.

12. Glossary

TermPlain-language meaning
HTMLThe language web pages are written in. .html files.
CSSThe language that makes web pages pretty (colors, fonts, layout).
JavaScript / JSThe language that adds interactivity (buttons, forms validating).
Repo / RepositoryA folder of code with full history of every change. Stored on GitHub.
CommitA saved snapshot of your changes with a descriptive message.
PushUploading your commits from your computer to GitHub.
BranchA parallel version of the code. We use one (main).
DeployPublishing the website so the internet can see the latest version.
CDN"Content Delivery Network" — copies of the site stored globally for fast loading.
DNSThe internet's address book — translates macrogen-es.com into a computer's location.
A recordA type of DNS entry that points a domain to an IP address.
CNAMEA DNS entry that points a subdomain (like www) to another domain.
HTTPS / SSLThe lock icon in the browser bar. Free and automatic on Vercel.
Static siteA website made of pre-written files (no database, no server-side processing).
VercelThe company that hosts our site for free.
GitHubThe company that stores our code with version history.
GoDaddyThe company that sold us our domain name.
Form actionWhere a form's submitted data goes. Currently nowhere (demo); will be Formspree.
FaviconThe small icon in the browser tab.
CTA"Call to action" — a button that asks the visitor to do something.
HeroThe big top section of a page (usually navy with the headline).
EyebrowThe small text label above a headline.
PillarA reusable card showing a value proposition.
Lorem ipsumPlaceholder Latin text used to mock up layouts.
