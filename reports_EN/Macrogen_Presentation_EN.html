Prompt 1 · Kickoff phase

Set the role + context + goal

Hey, I'm the CEO of Macrogen Chile. I need you to act as VP of Marketing, IT and Sales. [+ full context: company, services, global presence] Please build a Macrogen Chile website based on Academia Seoul. I have updated corporate identity. One team, one Macrogen. The site must include blog, newsletters, and different services. Research macrogen.com, macrogen-europe.com, dna.macrogen.com. — Original prompt, day 1
Why it worked Combines three things in a single prompt: (1) clear role assignment, (2) substantial business context up front (no need to ask 20 questions), (3) a specific deliverable. The reference URLs let me research the actual brand voice.

Prompt 2 · Fase de despliegue

Simple decisions, big impact

I prefer Vercel. — Short phrase, full decision
Step by step to make a demo version… it got tricky on Vercel because it wouldn't let me upload more than 20 files. — Describe the friction
Why it worked When something doesn't work, describe the friction. The hint about the 20-file limit let me immediately suggest the right fix: use GitHub instead of drag-and-drop.

Prompt 3 · Branding phase

Iterative refinement

Lets use the original Macrogen website [logo]. — Surgical change
Look up the other logos of Macrogen, we need to make this Macrogen logo Spain Portugal Chile Peru. — Regional expansion
Make all the order now CTA in the pink color for Macrogen to differentiate from the quotation. — System-wide change with a clear reason
Why it worked Each one is a small prompt that triggers system-wide changes. The intent is clear ("differentiate from the quote color") — that gives me a reason to apply the principle consistently across all 20 pages, not just fix one button.

Prompt 4 · Smart features

Specify behavior, not implementation

On the quote request, above the "name" field add a country selector — Spain, Portugal, Chile, Peru, other — so people pick their country and the info below auto-adapts: email TLDs like .pe / .es, and the WhatsApp section with the matching country code. — Specify the WHAT, not the HOW
For Spain, Portugal and others, route the email to info-spain@macrogen.com. For Chile and Peru, route it to info-chile@macrogen.com. — Explicit business rules
Why it worked Describe the experience the user should have ("the information converts automatically") instead of the technical implementation. I figured out how to wire the JavaScript + routing matrix from those two sentences alone.

Prompt 5 · Documentation

Ask for systems thinking

Can you create a logical process document so my team can see the site structure and the system? I want them to understand the engineering design process. — Specify audience + goal
The meta lesson for your team

The pattern across all these prompts: short, specific, with clear intent. None of them is a paragraph of technical specs. They're what you'd say to a smart colleague over coffee.

  1. State the desired outcome, not the implementation
  2. Provide context when it changes the answer (role, audience, constraint)
  3. Specify the "why" so the AI can apply the principle in many places
  4. Iterate fast — small prompts that build on prior work
